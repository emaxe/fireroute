# FireRoute

> API Gateway для LLM-инференса с ротацией ключей, групповым контролем доступа и аналитикой в реальном времени.

FireRoute (также известен как **FireGate**) — это прокси-шлюз для [Fireworks AI](https://fireworks.ai), обеспечивающий аутентификацию по Bearer-токенам, ротацию API-ключей, аналитику использования и полнофункциональную админ-панель на React.

---

## Возможности

| Возможность | Описание |
|-------------|----------|
| **OpenAI-совместимый прокси** | `/v1/chat/completions`, `/v1/embeddings`, `/v1/models` и другие |
| **Anthropic-совместимый прокси** | Эндпоинт `/v1/messages` |
| **Универсальный wildcard** | Любой эндпоинт Fireworks AI под `/v1/*` |
| **Ротация ключей** | Round-robin выбор API-ключа внутри группы |
| **Группы токенов (many-to-many)** | Один токен может быть привязан к одной или нескольким группам ключей |
| **Bearer-аутентификация** | Автоматическая маршрутизация запроса по привязанным группам |
| **Админ-панель** | Графики в реальном времени, фильтры, автообновление (Recharts + Tailwind) |
| **Аналитика генерации изображений** | Отдельный раздел с запросами, ошибками и латентностью |
| **Блокировка эндпоинтов** | Управляемый из админки чёрный список маршрутов |
| **Управление пользователями и токенами** | JWT-аутентификация, CRUD для пользователей, токенов, групп, ключей |
| **Логи запросов** | Полное логирование с токеном, ключом, группой, латентностью и подсчётом токенов |
| **Бинарные и SSE-ответы** | Изображения, аудио, видео и потоковые ответы передаются как есть |
| **Docker-стек** | Развёртывание одной командой `docker compose` |

## Быстрый старт

```bash
# Клонировать и перейти в папку проекта
git clone https://github.com/emaxe/fireroute.git
cd fireroute

# Скопировать переменные окружения и запустить всё
cp .env.example .env
docker compose up -d

# Gateway (API-прокси)      → http://localhost:9700
# Админ-панель              → http://localhost:9701
# Логин по умолчанию: admin@firegate.local / admin123
```

## Переменные окружения

Скопируйте `.env.example` → `.env` и отредактируйте:

| Переменная | Назначение | По умолчанию |
|------------|----------|--------------|
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql://firegate:firegate@postgres:5432/firegate` |
| `JWT_SECRET` | Секрет для подписи JWT | `change-me-in-production` |
| `ADMIN_EMAIL` | Email администратора по умолчанию | `admin@firegate.local` |
| `ADMIN_PASSWORD` | Пароль администратора по умолчанию | `admin123` |
| `GATEWAY_PORT` | Порт Gateway | `3000` |
| `FIREWORKS_BASE_URL` | Базовый URL upstream LLM API | `https://api.fireworks.ai/inference/v1` |

## Маршрутизация по группам токенов

Сервисный токен может быть привязан к **одной или нескольким** группам ключей. Прокси выбирает целевую группу в следующем порядке:

1. **Привязана 1 группа** → автоматическая маршрутизация в эту группу.
2. **Привязано N групп** → клиент обязан передать `"group": "group_id"` в теле запроса.
3. **Нет привязанных групп** → fallback на группу `default`.

> **Важно:** Прокси-роуты используют `preHandler` (не `onRequest`), чтобы Fastify успел распарсить тело запроса до проверки токена.

## Страницы админ-панели

- **Dashboard** — графики использования в реальном времени, потребление токенов, латентность, ошибки, статистика генерации изображений
- **Keys** — API-ключи с возможностью приостановки, привязка к группе, последнее использование
- **Groups** — группы ключей с round-robin ротацией
- **Tokens** — сервисные токены с множественным выбором групп
- **Users** — администраторы (JWT-аутентификация)
- **Logs** — логи запросов с фильтрами
- **Playground** — интерактивное тестирование прокси-запросов
- **Settings** — конфигурация шлюза (блокировка эндпоинтов и др.)

## Прокси-эндпоинты

| Эндпоинт | Совместимость | Методы |
|----------|--------------|--------|
| `/v1/chat/completions` | OpenAI | POST, GET |
| `/v1/embeddings` | OpenAI | POST |
| `/v1/models` | OpenAI | GET |
| `/v1/messages` | Anthropic | POST |
| `/v1/*` | Fireworks | Все |

Все прокси-эндпоинты требуют заголовок:
```
Authorization: Bearer <service_token>
```
